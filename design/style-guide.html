Visual Style Guide — Terrain v2 + Carbon Tech · March 2026

Design Principles

Strava DNA, not clone. White card surfaces, geometric sans body, Strava orange CTA. Warm page background + serif display + terracotta accent are the differentiators.

Color = awards. The UI is deliberately neutral so award pills are the brightest elements on screen — no competition from chrome.

Icons over dots. Geometric SVG outlines in each award's semantic color. Readable at 12px in pills, expressive at 32px in cards. Canvas-renderable for share images.

Type floor: 14px mono, 16px body. Target audience is 35+ cyclists reading on phones post-ride. DM Sans's wider letterforms help.

No gradients. Flat, solid colors throughout. The brand communicates through texture and typography contrast, not gradient effects.

Usage Rules

Terracotta Wordmark .io, highlighted data values in share cards, section accents, tagline Large background fills, buttons (use Strava for CTA)

Aluminum Share card dividers, subtle UI accents, secondary borders Primary text, backgrounds
Strava Orange "Connect with Strava" button only Everything else — this is Strava's brand, not ours
The monotonal trap: Terracotta and deep terracotta are the warm end. Steel blue and aluminum are the cool end. Use them together to create temperature contrast — warm header + cool share card frame, terracotta data highlights against steel structural elements.

Color — Award System

Awards are the only elements allowed to be colorful. Each award has four tokens: dot (icon/indicator), bg (pill fill), text (pill label), border (pill stroke). The neutral UI ensures these are always the brightest thing on screen.

Comeback Mode

Award colors are canonical. These values live in src/award-config.js and must not be changed without updating both the in-app rendering and the Canvas share card renderer. See src/icons.js for the matching SVG icon definitions.

Site Header

The header uses flat, solid deep terracotta — no gradients. The compact header on scroll uses the same color at the same value. The search bar area uses a slightly lighter treatment at reduced opacity.

Share Cards — Carbon Tech Treatment

Share cards need to pass the scroll-stop test on white social media feeds. The in-app UI is deliberately muted; the share card is where the brand needs to perform.

Construction

Canvas dimensions: 1080px wide, dynamic height. The card has three zones:

Implementation Notes

Share card renderer: The Canvas share card in ActivityDetail.js needs the Steel Blue header cap added. This is a Canvas fillRect at the top of the card, with the wordmark and card type rendered in --text-on-dark.

Header migration: Replace linear-gradient(135deg, #6B3518, #8E4A1F) in StickyHeader.js with flat #6B3518 for the main header and #8E4A1F for the search bar zone.

New CSS variables: Add --steel, --steel-light, and --accent-dark to app.css. The Steel variables are scoped to share cards and structural UI — they do not replace --border in the main app.

Scope: Carbon Tech (steel blue) applies to share cards and structural accents only. The in-app card experience remains white-on-warm-paper with the existing border palette. Don't mix share card aesthetics into the main UI.
